Politique de confidentialité — MoodTrack

Dernière mise à jour : 2025-10-21

1. Présentation

Cette application mobile ("MoodTrack") permet de suivre des habitudes, l’humeur et un journal personnel. Cette politique explique quelles données sont collectées, comment elles sont utilisées et les choix qui s’offrent à vous.

2. Données stockées localement

Les données suivantes sont stockées localement sur l’appareil :

  • Liste d’habitudes (nom, activation, rappels)
  • Journaux quotidiens (checks d’habitudes) et entrées d’humeur
  • Notes / journal
  • Paramètres (thème, préférences, état premium)

Le stockage se fait via SQLite et SharedPreferences. Aucune de ces données n’est envoyée au serveur par défaut.

3. Publicité et consentement

L’application utilise Google Mobile Ads (AdMob) pour afficher des bannières et interstitiels. Pour les utilisateurs de l'EEE, un formulaire de consentement (UMP) est utilisé afin de demander l’autorisation pour la publicité personnalisée. Si vous refusez, seules des annonces non personnalisées peuvent être affichées.

Identifiants publicitaires et configuration : com.google.android.gms:play-services-ads et UMP.

4. Achats intégrés

Les achats d'abonnement et d’option "no ads" sont gérés via le magasin (Google Play / App Store) avec le plugin in_app_purchase. Les informations d’achat (reçus) sont traitées par le Store ; l’application enregistre localement l’état premium pour débloquer les fonctionnalités.

5. Données tierces et analytics

Actuellement l’application n’envoie pas de données d’usage à un serveur externe pour de l’analytics. Les SDK tiers (AdMob, UMP, IAP) peuvent collecter des informations nécessaires au fonctionnement (identifiants d’appareil pour la diffusion d’annonces, état d’achat). Consultez les politiques de ces services pour plus de détails.

6. Partage et export

Vous pouvez exporter vos données (backup JSON/CSV) et les partager manuellement via l’interface de l’app. Ces fichiers contiennent les données stockées localement. Le partage est effectué par l’appareil (share sheet) et non transmis automatiquement à des services tiers par l’application elle‑même.

7. Sécurité

Les données restent sur votre appareil sauf si vous choisissez de les exporter ou d’utiliser un service tiers externe. Aucune mesure de chiffrement avancée n’est appliquée par défaut au contenu des fichiers locaux ; protégez l’accès physique à votre appareil.

8. Conservation

Les données sont conservées indéfiniment sur l’appareil jusqu’à votre suppression ou jusqu’à la réinitialisation de l’application. Les backups exportés restent sous votre responsabilité.

9. Vos droits

Vous pouvez :

  • Supprimer vos données en désinstallant l’application ou via l’option d’effacement si fournie.
  • Restaurer des achats via le bouton "Restaurer" dans les paramètres si vous changez d’appareil.

10. Contact

Pour toute question au sujet de cette politique ou pour demander la suppression des données, contactez : contact@exemple.com

11. Modifications

Nous pouvons mettre à jour cette politique. La date de "Dernière mise à jour" en tête du document indique la version active. Nous recommandons de vérifier cette page avant d’utiliser l’application après une mise à jour.

Remplacez les champs entre crochets et l’adresse de contact par vos informations réelles avant publication. Pour une conformité complète (GDPR, CCPA), envisagez une revue légale adaptée.
